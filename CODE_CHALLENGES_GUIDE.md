## Accessing Code Challenges

### From the UI
1. Click the **Target** icon (cyan gradient button) in the top toolbar
2. Browse challenges by category or difficulty
3. Click a challenge card to start coding
4. Write your solution and click "Run Tests"

### From Tutorials
1. Watch video tutorials in the Tutorial panel
2. Look for the **Practice** button on tutorial steps
3. Click to jump directly to related challenges
4. Apply what you learned immediately

- `Ctrl/Cmd + Shift + C` - Open challenges panel

## Challenge Structure

Each challenge includes:

### 1. Challenge Card
- **Title**: Clear, descriptive name
- **Description**: What you'll build
- **Difficulty Badge**: Beginner, Intermediate, or Advanced
- **Time Estimate**: Expected completion time
- **XP Reward**: Points earned for completion
- **Category Icon**: Visual category indicator
- **Completion Status**: Checkmark for completed challenges

### 2. Learning Objectives
- Clear list of concepts you'll practice
- Skills you'll develop
- Related programming patterns

### 3. Code Editor
- **Starter code**: Pre-populated template
- **Syntax highlighting**: Easy-to-read code
- **Auto-save**: Never lose your progress
- **Reset option**: Start over if needed
- **Insert to editor**: Copy solution to main editor

### 4. Test Cases
- **Public tests**: Visible requirements
- **Hidden tests**: Edge cases for thorough validation
- **Input/Output examples**: Clear expectations
- **Detailed error messages**: Learn from mistakes

### 5. Hints
- **Progressive difficulty**: Start with gentle nudges
- **Code examples**: Visual learning aids
- **Conceptual guidance**: Understand the "why"
- **Penalty system**: Encourages independent problem-solving

## Technical Implementation

### Test Execution
- **Safe evaluation**: Code runs in isolated context
- **Multiple test cases**: Comprehensive validation
- **Error handling**: Graceful failure messages
- **Performance**: Fast execution with visual feedback

### Data Persistence
- **Progress saved**: useKV for automatic sync
- **Current code**: Resume where you left off
- **Stats tracking**: Historical data preserved
- **Cross-session**: Access from any device

### XP Calculation
```javascript
Base XP = Challenge reward (10-75 XP)
Bonus XP = Base * 0.2 (if no hints used)
Total XP = Base + Bonus
```

### Streak Calculation
- Increments when completing within 24 hours of last completion
- Resets if more than 1 day gap
- Maintains motivation for daily practice

## Troubleshooting

### Tests Not Running
- Check for syntax errors in your code
- Ensure function name matches starter code
- Verify browser console for errors

### Progress Not Saving
- Check useKV is functioning
- Verify browser storage permissions
- Try refreshing the page

### Hints Not Revealing
- Click the "Reveal Hint" button
- Ensure you're viewing the correct hint level
- Check for JavaScript errors

### Code Insert Not Working
- Make sure a file is open in the editor
- Create a new file if needed
- Check that activeTab exists

## Contribution Guidelines

### Adding New Challenges
1. Create challenge object in `/src/lib/challenge-data.ts`
2. Include all required fields:
   - id, title, description
   - difficulty, category, language
   - starterCode, solution
   - hints array, testCases array
   - learningObjectives
3. Write comprehensive test cases
4. Provide progressive hints
5. Test thoroughly before committing

### Challenge Quality Standards
- Clear, concise descriptions
- Realistic time estimates
- Fair XP rewards based on difficulty
- At least 3 test cases (1 hidden)
- 2-3 progressive hints
- Proper starter code structure
